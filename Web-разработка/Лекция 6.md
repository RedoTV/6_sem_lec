\[\[prototype]] или \_\_proto__  - резервное хранилище свойств и методов объекта, автоматически используемое при поиске. 

\_\_proto__ 

```js
var foo = {
	x: 10,
	y: 20
};

var foo = {
	x: 10,
	y: 20,
	[[prototype]] : Object.prototype
}
```

почти все объекты в js имеют объекты прототипы от которых наследуют свойства и методы. Все объекты, созданные с помощью литералов имеют один и тот же объект прототип, на который в js скрипте ссылаются Object.prototype. У него нет прототипа, т.е. нет унаследованных свойств. 

```js
Object.prototype = 10; //monkey patching

var a = {}; //create "empty" "hash"

alert(a["x"]); //10, but it's not empty 
alert(a.toString); //function
alert(a.constructor); //function
```

Обезьяний патч - это возможность подмены методов и значений аттрибутов классов скрипта во время его выполнения. Практическое применение техники обезьяних патчей - расширение функционала чужих продуктов (фреймворков, библиотек и т.д.), без непосредественного вмешательства в их код. 

Исключаем свойство, непринадлежащее самому объекту 

```js
var emptyHash = Object.create(null);
console.log(emptyHash.toString); //undefined
```

объект, созданный при помощи `Object.create(null)` не имеет прототипа, а значит в нем нет лишних свойств.

```js
// monkey patching
Object.prototype.bar = 1;

var foo = {moo: 2};
for(var i in foo) {
	console.log(i); //bar, moo
}

for(var i in foo) {
	if(foo.hasOwnProperty(i)) {
		console.log(i); //moo
	}
}
```

Обычный цикл for .. in не делает различия между свойствами объекта и его прототипа, он перебирает всё.
