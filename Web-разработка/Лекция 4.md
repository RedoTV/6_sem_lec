### 3. Scope. Область видимостей

В ES3 существует только 2 типа областей видимости:
- Глобальная - область видимости всей веб страницы или node js приложения
- Функциональная - область видимости разная у каждой функции.

Объекты относятся к глобальной области видимости если: 
1. они определены вне области действия функции
2. они определены без var

#### *shadowing* (затемнение или экранирование) - недоступность одноименной внешней переменной внутри функции.

#### *hoisting* (поднятие или всплытие) - js поднимает все объявления переменных, перемещая их в начало их прямых областей видимости.

```js
var scope = "global";
function f() {
	alert(scope); // undefined
	var scope = "local";
	alert(scope); // local
}
f();
```

интерпретатор объявляет сам, потому что всплывает имя var переменной но не ее значения:
```js
var scope = "global";
function f() {
	var scope;
	alert(scope); // undefined
	scope = "local";
	alert(scope); // local
}
f();
```

```js
var foo = 1;

function bar() {
	if(!foo) { // foo = undefined, !foo сработает в if
		var foo = 10;
	}
	alert(foo); // 10
}
```

Все скрипты подключенные на страницу имеют общую область видимости.

В JS все глобальные переменные функции являются свойствами глобального объекта, в браузере он доступен под именем window. Для доступа к текущему объекту из метода используется ключевое слово this.

### Порядок инициализации

Выполнение скрипта происходит в 2 фазы: 
1. инициализация, подготовка к запуску.
   Во время инициализации скрипт сканируется на предмет объявления функции вида Function Declaration (стандартное объявление функции), а затем - на предмет объявления var переменных. каждое такое объявление добавляется в window. Function Declarations создаются сразу работающими, а var переменные - равными undefined.
2. Присваивание значений переменных происходит, когда поток выполнения доходит до соответсвтвующей строчки кода, до этого они - undefined.

function f() {
	x = "global variable";
}
f();
this.x === "global variable";
window.x === "global variable"; // true for browsers

В node.js для доступа к глобальному объекту используется переменная global.

let переменные нельзя повторно объявлять.

### Closures (замыкания)

Замыкание - функция вместе со всеми внешними переменными, которые ей доступны.

Все переменные внутри функции - это свойства специального внутреннего объекта LexicalEnvironment. При запуске функции он создается, в него записываются аргументы функции и переменные.

Каждая функция при создании получает ссылку scope на объект с переменными, в контексте которого была создана. При запуске функции создается LexicalEnvironment, он получает ссылку на внешний объект переменных из $[[Scope]]$ при поиске переменных он осуществляется сначала в текущем LexicalEnvironment, а потом по этой ссылке. Если переменная не найдена во внешнем LexicalEnvironment, то она ищется в еще более внешнем (через $[[Scope]]$ внешней функции).

```js
function foo() {
	//LexicalEnvironment = {x : undefined}
	var x = 'Hello';
	// LexicalEnvironment = {x : Hello}
	
	function bar() { // local function
		console.log(x);
	};
	return bar;
}

var func = foo();
func();
```

В примере внутри одной функции создается другая и возвращается в качестве результата. Вызов foo() завершается и функция bar возвращается и сохраняется во внешней переменной func;

```js
function userThis(name) {
	this.myName = name;
	
	function returnMe() {
		return this;
	}
	
	return {
		returnMe : returnMe
	}
}

var userthis = new userThis('Julia');
alert("UsesThis thinks it's called" + usesThis.returnMe().myName); 
// UserThis thinks it's called undefined
 
```

У каждой классической функции свой this.

Значением this является объект перед точкой, в контексте которого вызван метод. Чтобы получить доступ к this внешней функции можно сохранить его в переменной

```js
function userThis(name) {
	var that = this;
	this.myName = name;
	
	function returnMe() {
		return that;
	}
	
	return {
		returnMe : returnMe
	}
}

var userthis = new userThis('Julia');
alert("UsesThis thinks it's called" + usesThis.returnMe().myName); 
```


Если функция использует this - это подразумевает работу с объектом. при использовании this без привязки к контексту вызова в ES3 возвращается Window, в строгом режиме - undefined (ES6).

IIFE (Immediately-invoked function expression) мгновенно выполняемое функциональное выражение. это JS шаблон проектирования, который создает лексическую область видимости, используя JS-функциональные области видимости. 

```js
(function () {...} ) ();
(function () {...} ()); // douglas crockford's style. - означает немедленнный вызов функционального выражения, на месте разрешено вызывать только function expression, поэтому чтобы не было ошибки поэтому тело функции нужно взять в круглые скобки. 
```

```js
var v, getValue;
v = 1;
getValue = function() {
	return v;
};
v = 2;
getValue(); //2
```

как сохранить для использоваения старое значение переменной:
1. IIFE
```js
var v, getValue;
v = 1;
getValue = (function(x) {
	return function() { return x; };
})(v);
v = 2;

getValue(); //1
```

Функция передает v как аргумент и сразу исполняется сохранив контекст выполнения внутренней функции. IIFE создает локальную область видимости, т.е. запоминает старое значение переменной.

2. Использование замыканий
```js
var v, getValue;
v = 1;

function f(x) {
	return function() { return x; };
}

getValue = f(v;)
v = 2;

getValue(); //1
```

функция work() получает значение name из того места где была создана, через ссылку на внешнее лексическое окружение, а функция makeWorker() в момент создания запоминает свой let name в lexical environment.
