### 1. Операторы 

Оператор или (||) вычисляет ровно столько значений, сколько необходимо до первого true. При этом оператор или возвращает то значение, на котором остановились вычисления, причём не преобразованное к логическому типу. 

Оператор и (&&) вычисляет слева и справа выражение до первого ложного и возвращает его, а если все истинные - то последнее значение.

Приоритет оператора И больше чем ИЛИ. 

alert(5 || 1 && 0); // 5

Если перед или после символа плюс находятся не строки и не числа. JS автоматически преобразует операнды либо к строкам либо к числам. boolean, null, undefined преобразуются к числам.

True == 1, false == 0, null = 0, undefined == NaN.

Объекты, массивы и функции преобразуются к строкам.
[] + [] (js преобразует оба массива и сложит получившиеся 2 строки) при преобразовании массива в строку, получается строка, содержащая список его элементов через запятую. 

Оператор instanceof используется для проверки, принадлежит ли объект заданному типу. В левой части указывается проверяемый объект, в правой - функция-конструктор для проверки.

При сравнении значений разныз типов осуществляется численное преобразования для проверки равенства без преобразования типов используются операторы

alert( 0 === false ); false

alert( null > 0 ); // false
alert( null == 0); //false

alert (null >= 0); //true

в 1-ом и 3-ем примерах при сравнении разных типов происходит численное преобразование. null -> 0. Во втором примере численное преобразование не происходитит, т.к. null дружит только с undefined. 

alert( undefined > 0 ); // false
alert( undefined < 0); //false
alert (undefined == 0); //false

В 1-ом и 2-ом примерах происходит численное преобразование (undefined -> NaN, NaN по стандарту устроено так что любые сравнения с ним возвращают false).

В 3-ем примере undefined дружит только с null.
