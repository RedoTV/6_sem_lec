### Функции в JS

Основные отличия function declaration и function expression:
1. Функции, объявленные как function declaration создаются интерпретатором до выполнения кода, поэтому их можно вызвать до объявления.
2. function expression создаются в процессе выполнения выражения, в котором созданы, до этого их вызвать нельзя
3. условные объявления if работают только с function expression 

В function declaration при 'use strict' видны только внутри блока, в котором описаны.

функциональные выражения нельзя вызывать до объявления. Имя именованного функционального выражения доступно только внутри самой функции и используется для рекурсии.

$new Function(params, code)$ - конструктор функции, так можно создать функцию код которой неизвестен  на момент написания программы, но строка с ним генерируется или подгружается во время её выполнения.
```js
variableName = new Function(
	/*
		variables
	*/
	,
	functionalBody
)
```

```js
function func(obj) {
	obj.prop1 = 5;
	obj = {};
}

var data = {};

func(data);
console.log(data); // {prop1: 5}
```

параметры копируются в локальные переменные функции. В obj и data ссылка на 1 и тот же объект.

Assert - если не выполняется выражение внутри - прекращает выполнение скрипта.

Когда функции требуется статическая переменная, значение которой должно сохраняться между вызовами, удобно использовать свойство объекта функции, позволяющее не занимать пространство имен определениями глобальных переменных. 

```js
function getNewId() {
	if(!getNewId.lastId) getNewId.lastId = 0;
	return ++getNewId.lastId;
}
```

если функция не вернула значение или return был без аргументов, то считается что функция вернула undefined, за исключением конструкторов у которых значение, возвращяемое по умолчанию -  this. 

Стратегии передачи аргументов в функцию:
1. Если функция вызывается со слишком большим количеством аргументов, то лишние аргументы игнорируются. 
2. Если функция вызывается с недостаточным количеством аргументов, то отсутствующие значения будут заменены на undefined.

Для задания значения по умолчанию можно использовать 2 способа:
1. можно проверить равен ли аргумент undefined (\=\=\=), и если да тогда значением по умолчанию  
2. использовать оператор или (||).


Если функция имеет более 5-ти параметров, то становиться трудно запоминать порядок исследования. 

Можно предусмотреть возможность передачи аргументов в виде пар имя:значение в произвольном порядке, чтобы реализовать такую возможность, при определении функции следует предусмотреть передачу объекта в виде единственного параметра.

Если число аргументов в вызове функции превышает число имен параметров, то функция лишается возможности напрямую обращаться к неименованным значениям. Решение этой проблемы - объект Arguments. В теле функции идентификатор arguments ссылается на объект Arguments, присутствующий в вызове. Объект Arguments - массивоподный объект, позволяющий извлекать переданные функции значения по их номерам. 

arguments содержит следующие свойства:
- callee - ссылка на текущую функцию
- length - количество реально переданных аргументов
- properties-indexes - параметры вызова.

Если вызову функции или метода предшевствует ключевое слово `new`  то это вызов конструктора. Если конструктор явно вернет объект с помощью конструкции return, то значением выражения вызова конструктора станет этот объект. Если конструктор выполнит return без значения или вернет с его помощью простое значение, то это значение будет проигнорировано и в качестве результата вызова будет использован новый объект.

Если функция создается с помощью конструктора new Function то не учитывается лексическая область видимости. Функции всегда компилируются как глобальные. при каждом вызове конструктор Function выполняет синтаксический анализ тела функции и создает новый объект функции. Методы функции `call` и `apply`   позволяют выполнить косвенный вызов функции, как если бы она была методом некоторого другого объекта. Первым аргументом обоим методам передается объект, относительно которого вызывается функция, этот аргумент определяет контекст вызова - `this`.

Чтобы вызвать функцию `f()` без аргументов, как метод объекта `o`:

```js
f.call(o);
f.apply(o);
```

Любой из этих способов эквивалентен следующему коду

```js
o.m = f; //временно сделать f методом o
o.m(); // вызывать его без аргментов
delete o.m; // удалить временный метод
```

Все остальные аргументы методом, следующие за первым аргументом передаются вызываемой функции:

```js
f.call(o, 1,2);
f.apply(o, [1,2]);
```

Метод toString() возвращает строку вида `[object class]`. Для правильного определения класса объекта вызываем метод toString() этого объекта и извлекаем из результата класс (подстроку с 8-го по предпоследний символ).

При помощи `call` или `apply` можно взять метод 1-го объекта, в том числе и встроенного, и вызвать в контексте другого. arguments - массивоподобный объект, методов для работы с массивами у него нет.

```js
function f() {
	var args = [].slice.call(arguments, 1, 3);
	
	return args;
}

f(1,2,3,4,5,6); // [2,3];
```

Такой вызов технически возможен, потому что slice для работы требует только нумерованные свойства и length, всё это у arguments есть.

Чтобы вернуть объект обернем тело функции в круглые скобки. Чтобы получить массив аргументов можно использовать arguments или оператор rest. В `rest` попадет массив всех аргументов, начиная с 3-го. 

`rest` - массив с соответствующими методами. Оператор `...` должен быть в конце так как он собирает оставшиеся аргументы.

Утиная типизация - стиль динамической типизации, в которой текущий набор методов и свойств объекта определяет допустимую семантику. Утиная типизация - проверка реализациям объекта требуемого интерфейса. Смысл утиной типизации - проверки необходимых методов и свойств. Мы можем проверить что объект - массив, не вызывая `Array.isArray` , а просто уточнив наличие важного для нас метода (splice)

```js
var t ={
	"0" : "a",
	"1" : "ab",
	"2" : "abc",
	length:3

[].push.call(t, "abcd");

```

Значение на котором заканчивается рекурсия называется базисом рекурсии. Общее количество вложенных вызовов называют глубиной рекурсии. У каждой рекурсии есть свой контекст выполнения - это служебная информация соответствующая текущему запуску функции. Она включает в себя локальные переменные функции. При любом вложенном вызове интерпретатор запоминает текущий контекст выполнения в специальной структуре данных - стеке контекста. Затем интерпретатор приступает к выполнению вложенного вызова. Для нового вызова создается свой контекст выполнения. И управление переходит в него, а когда он завершен - старый контекст достается из стека и выполнение внешной функции возабновляется. 

Хвостовая рекурсия - интерпретатор может понять, что текущий вызов функции в стеке стереть его и заменить его следующим вызовом. Рекурсия не приводит к разрастанию стека. Главное условие хвостовой рекурсии: вызов функции и возврат ее результата должны быть последними действиями в текущей функции.

```js
function factorial(n) {
	function recur (n, acc) {
		if(n == 0) {
			return acc;
		} else {
			return recur(n-1, n*acc);
		}
	}
	return recur(n, 1);
}
```

Оптимизация хвостового вызова (рекурсии) означает чтто мы не делаем полноценный вызов функции включая создания уровня в стеке. Мы запоминаем текущее значение факториала во втором параметре и текущий вызов в стеке нам уже не нужен.

Трамплин - функция, которая итеративно вызывает возвращённую функцию-преобразователь. Когда мы вызываем функцию, она возвращает преобразователь, которую мы вызываем, чтобы получить результат. Но преобразователь может вернуть еще 1 преобразователь, и так каждый раз, когда мы получаем результат, мы проверяем не преобразователь ли он. Если нет, то мы получили остаточный (конечный) результат (Декоратор или обертка в python). 

Метод bind связывает функцию с объектом. Если вызвать метод `bind` в функции `f`  и передать ему объект `o`, то он вернет новую функцию. Вызов новой функции (как обычной функции) выполнит вызов оригинальной функции `f` как метод объекта `o`. Любые аргументы, переданные новой функции, будут переданы оригинальной функции.
