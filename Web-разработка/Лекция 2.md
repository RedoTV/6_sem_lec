### 8. Работа с массивами в JS

Методы массивов:
- concat - объединяет 2 массива в 3-ий - используя concat можно передавать в массив значения
- join - преобразует все элементы массива в строки, по умолчанию - (,) 
- shift - удаляет первый элемент массива, смещая все последующие элементы на 1 позицию влево, возвращает удаленный элемент.
- pop - удаляет последний элемент массива и возвращает его.
- unshift - добавляет в начало массива указанное значение (буква я) и возвращает получившуюс я длину
- push - добавляет в конец массива указанное значение ('я') и возвращает получившуюся длину.
- reverse - изменяет порядок следования элементов массива на противоположный.
- sort - лексикографически сортирует массив
- slice - копирует участок массива от идекс1 до индекс2 не включая индекс2, исходный массив при этом не изменяется (.slice(индекс1\[,индекс2])) . Если не указать индекс2 - то копирование будет до конца массива. Можно использовать отрицательные индексы, если не указать аргументы - то скопируется весь массив. если индекс1 < индекс2, то будет создан пустой массив.
- splice(индекс, количество) - 
	  
	  Используется для удаления элементов из массива, вставки и замены элементов. При удалении первый параметр - индекс с которого начинать удаление, второй параметр - количество удаляемых элементов, начиная с указанного индекса. 
	  Для добавления новых элементов в массив в качестве первого параметра указываем номер индекса, начиная с которого будем добавлять элементы, в качестве второго параметра указываем 0. 0 говорит методу о том, что никакие элементы удалять не нужно. Далее через запятую перечисляем новые элементы. При замене элемента в методе splice: 1-ый параметр - номер индекса, 2-ой - количество заменяемых элементов, далее через запятую сами заменяющие элементы.
	  
- toString() - преобразует содержимое массива в строку.


### 9. Основные операторы JS

async - скрипт выполняется полностью асинхронно, т.е. при обнаружении скрипт src браузер не останавливает загрузку страницы, а работает дальше. Когда скрипт будет загружен - он выполниться. defer - скрипт так же выполняется асинхронно, не заставляя ждать страницу, есть 2 отличия от async.
```js
<script src="1.js" async></script>
<script src="2.js" defer></script>
```

Отличия defer от async:
1. браузер гарантирует, что относительный порядок скриптов с defer будет сохранен, т.е. в коде с async первым сработает тот скрипт, который загрузиться раньше. В коде с defer всегда первым сработает script 1.js, а script 2.js даже если загрузился раньше, будет его  ждать. Поэтому defer используют в тех случаях, когда 2-ой скрипт зависит от 1-го.
2. скрипт с defer сработает только тогда, когда весь html документ будет обработан браузером.

async и defer работают только со внешними скриптами (нужен src). 

Теги script можно добавлять на страницу при помощи js в любой момент. Такие скрипты ведут себя как async. 

Чтобы перевести код в режим полного соответствия современному стандарту языка в начале функции или скрипта нужно указать директиву 'use strict'; или "use strict";
